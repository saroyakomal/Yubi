YMPL Mutual Funds — UX Structure (Phase 1)

Scope: In-house Mutual Funds module for YMPL distributor app. Replaces the current Investwell SSO redirect.
Audience: Designer, frontend & backend developers.
Parity reference: Existing Fixed Deposits module — distributor-facing, client-selection-based. MF should feel identical in tab labels, navigation, and flow patterns.

Mandate notes for developers:
  • UMRN = Unique Mandate Reference Number (NPCI-issued, 20 chars, immutable).
  • Mandate is a real banking instrument at NPCI — used by Razorpay/BSE Star to physically pull money on each SIP date.
  • Investor authorises via net banking / debit card + ATM PIN / Aadhaar OTP. Bound to a specific bank account.
  • Credit cards are not allowed for MF SIP debits per SEBI/AMFI rules.
  • One mandate can serve multiple plans within its max-amount cap.
  • STP has no mandate (intra-RTA, no bank debit).
  • SWP has no mandate (outflow to investor's bank, not debit from it).

4. Setup STP

Locked.

Entry points

PriorityEntryNotes
PrimaryInvestor Profile → Portfolio → Holdings row → "Setup STP"Source = folio. No order-to-folio resolution.
Secondary (shortcut)Orders tab → row action "Setup STP from this scheme"Resolves the order to its folio, lands on the same setup page.
Do not place STP entry in Explore — Explore is for discovering new funds, not transferring out of existing holdings.

Setup STP page (dedicated full page)

Source (locked, pre-filled)

Client · PAN (masked) · Folio · Scheme · Plan/Option · Current units · Current value · Exit load status

Target scheme

Searchable picker (same AMC for intra-AMC STP) · Target plan (Direct/Regular) · Target option (Growth/IDCW)

Transfer plan

  • Amount per cycle (₹)
  • Frequency: Monthly / Quarterly
  • Transfer day (1–28)
  • Start date
  • End condition: End date / Number of installments / Until source exhausted

Disclosure

Acknowledgement: "Each STP cycle is a redemption from source + purchase in target. Capital gains apply on the source leg per slab/LTCG rules. Exit load may apply if source units are below threshold age."

Pre-flight check (system, automatic)

  • Validates: folio active, current value above STP minimum.
  • If folio has an active SIP feeding in, shows warning: "This folio has an active SIP. Continuing may cause churn. Pause SIP?" — distributor can pause SIP inline or proceed.

Footer actions

Cancel · Submit

End-to-end flow

  1. Distributor opens Investor Profile → Portfolio → Holdings.
  2. Clicks "Setup STP" on the source folio row.
  3. Pre-flight check runs (active-SIP warning if applicable).
  4. Setup page opens — source pre-filled and locked.
  5. Distributor picks target scheme, fills transfer plan, acknowledges tax disclosure.
  6. Submits → Plan created with Status = Active.
  7. New row in Systematic Plan table. No mandate row. No order row yet.
  8. On each cycle date: AMC executes redeem + buy → one Switch order row in Orders tab. Source folio shrinks; target folio grows.
  9. Lifecycle: Pause / Modify / Stop from Systematic Plan section. Auto-completes when end condition met OR source exhausted.
  10. Original buy/SIP orders untouched throughout.

Status lifecycle

Draft → Active → [Paused] → Completed / Stopped.

Disclosure

Acknowledgement: "Each SWP cycle is a redemption; capital gains tax applies per slab/LTCG rules."

Footer actions

Cancel · Submit

End-to-end flow

  1. Distributor opens Investor Profile → Portfolio → Holdings.
  2. Clicks "Setup SWP" on the source folio row.
  3. Pre-flight: validates current value above SWP minimum, exit-load warning if redeeming within window.
  4. Setup page opens — source and payout bank pre-filled and locked.
  5. Distributor sets withdrawal type, amount/units, frequency, schedule, end condition.
  6. Acknowledges tax disclosure → Submit.
  7. New row in Systematic Plan table. No mandate row. No order row yet.
  8. On each cycle date: AMC redeems → proceeds credited to investor's KYC bank (T+2/T+3) → one Redeem order row in Orders tab. Source folio shrinks.
  9. Lifecycle: Pause / Modify / Stop. Auto-completes when end condition met OR source exhausted.

Status lifecycle

Draft → Active → [Paused] → Completed / Stopped.

Cross-cutting design notes

  • Naming consistency: use the exact FD tab labels (Home / Explore / Orders) — not "Dashboard", not "Funds", not "Transactions".
  • Buy → Lumpsum: rename tab entry to Invest (same verb FD uses on the card). Keep the two-step pattern of FD (Detail → Review).
  • Select Bank screen: not needed as a standalone screen if the investor has only one bank on record (which is the Phase 1 default). Show inline on Review. Promote to a full step only when multiple banks exist in a later phase.
  • Configure SIP: reuses the Setup SIP screen with prefilled + editable values. "Modify" reopens the same form.
