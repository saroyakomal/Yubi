MF Phase 1 UX structure — locked

All 7 designer "Komal" points addressed and finalised. Single source of truth published at MF/mf-structure.html.

  • Phase 1 decisions: Commission out of scope · Free for investor · SIP full-edit · Redemption distributor-initiated only.
  • Top-level navigation: Home deferred to Phase 2+ (lands directly on Explore); Explore · Orders · Portfolio locked.
  • Explore tab: Switched from card grid to list view (parity with Groww / Zerodha Coin / Kuvera / ET Money). 7 columns: Scheme · Category · Returns (1Y/3Y/5Y) · AUM · Expense ratio · Riskometer · Min SIP. Distribution fee shown as "Coming soon" (FD parity). Search bar + 10 filters defined. Default sort: 3Y CAGR descending.
  • Orders tab: Columns finalised — App ID · Client · Scheme · Folio Number · Type · Txn type · Amount · Units · NAV+date · Status · BSE Star order ID · Ordered at · Plan ID. View Detail spec defined (Header / Client & scheme / Order table).
  • Portfolio (inside Investor Profile): Holds Holdings table + Systematic Plans table + AUM summary. Investor profile is opened from Client page → click investor name.
  • Holdings table: One row per folio. 11 columns including Avg buy price, Current NAV, Unrealized P&L. Row actions: Buy more / Setup SIP / Setup STP / Setup SWP / Redeem / View transactions. This is the primary entry point for STP and SWP setup.
  • Systematic Plan + Mandate (unified): Single hierarchical table — parent row = plan (SIP/STP/SWP), expandable child = Mandate Debit History (one row per debit attempt, 13 columns: Date, UMRN, Scheme, Linked plan, Status, Amount, Bank, Account masked, Max amount, Frequency, Start/End, Auth mode, Failure reason). Mandate is no longer a separate tab.
  • Setup STP: Primary entry from Holdings; secondary shortcut from Orders. Dedicated full page. Source folio locked, target scheme picker, transfer plan, tax disclosure, review & submit. Status lifecycle Draft → Active → [Paused] → Completed/Stopped.
  • Setup SWP: Primary entry from Holdings; secondary from Orders. No mandate row (outflow to investor's bank, not a debit). On execution, AMC redeems and credits investor's KYC bank T+2/T+3.
